In Ruby, Hash#values returns a new array of the receiver's values.

Like any other array, vals contains pointers to values, not the values themselves. So it sees mutations to those values, no matter if the mutations are effected via h. Apart from pointing at the same values, it is as independent of h as any other array. So it does not see values added to h.

In Python, dict.values() returns a view object: a dynamic proxy of the dictionary's values.

Sure, vals sees mutations to those values. But so would [d[k] for k in d], since, like Ruby, lists contains pointers to values, not the values themselves. What makes it dynamic is that it sees values added to d too.
